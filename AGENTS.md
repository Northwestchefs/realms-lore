# AGENTS.md — realms-lore

## Core Rules (read first)
- Keep this repo a **Forgotten Realms lore vault** for **Obsidian + Quartz**.
- Prefer **scaffold-first pages**: short, structured, expandable.
- Use **wiki links** (`[[path/page|Label]]`) and clean, Quartz-friendly Markdown.
- Avoid bloated essays; prioritize navigable reference notes.
- Preserve existing folder structure; do not rename/move/delete established content without strong reason.

## Editorial Scope
- Write player-safe, campaign-usable lore unless a page explicitly says otherwise.
- Optimize for quick table use now and deeper expansion later.
- Favor reusable sections, cross-links, and hub/index discoverability over dense prose.

## Markdown & Linking Conventions
- Use plain Markdown with clear heading hierarchy (`#`, `##`, `###`).
- Prefer short sections and bullets over long paragraphs.
- Link related pages proactively with Obsidian-style wiki links.
- Keep formatting simple and stable for Quartz rendering.

## Canon Ambiguity & Timeline Handling
- Be careful with disputed canon (rulers, dates, edition conflicts, exact chronology).
- If canon varies by source/era, use broad, campaign-friendly phrasing.
- Do not present uncertain details as settled fact.
- It is acceptable to leave pages intentionally light when certainty is low.

## Content Boundaries by Topic Type
- **nations-realms/**: high-level identity, government tendencies, major factions, key relationships.
- **regions/**: geography, political context, travel/use-at-table framing, notable sub-areas.
- **settlements/**: local governance, districts/landmarks, economy/culture hooks, immediate adventure value.
- Keep level-appropriate detail in each layer; avoid duplicating full content across nation/region/settlement pages.

## Structure Preservation
- Keep existing architecture intact (nations-realms, regions, settlements, faiths, factions, people, noble-houses, landmarks).
- When expanding a broad topic, prefer adding related stubs over overloading one page.
- Maintain or improve hub pages for navigation and backlink health.

## Workflow Expectations for Codex Runs
- Make focused, logical commits with clear messages.
- Open a PR for completed work with a concise summary and validation notes.
- Avoid unrelated file edits unless needed for consistency.
